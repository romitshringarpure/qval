LAYOUT
- Package qval/, console script "qval = qval.cli:main" (argparse dispatcher in qval/cli.py). Subcommands in qval/commands/: init.py, doctor.py, run.py, import_cmd.py, gate_cmd.py, report_cmd.py — all six are real. stubs.py is the seam for registering not-yet-built commands (they would print "not implemented — ships in F-XX" and exit 3); its _STUBS registry is EMPTY today. - qval/engine/: schemas.py = NATIVE types: TestCase, ModelResponse, DetectorResult, TestResult, RunSummary. Native statuses are UPPERCASE "PASS"/"FAIL"/"NEEDS_REVIEW"; native risk levels critical/high/medium/low.
- qval/engine/model_client.py: ModelClient Protocol = {provider: str, complete(prompt) -> ModelResponse}. Implementations: OpenAIClient (openai SDK, retry via _retrying_call with exponential backoff), AnthropicClient (requests-based STUB — raises, shows the wire-up shape), MockClient (deterministic, seeded). Client construction happens in qval/commands/run.py build_client(config, mock=..., model_override=..., ...).
- qval/engine/test_runner.py: TestRunner(run_id, client, ...).run(cases, suite_label) -> (list[TestResult], RunSummary). RunSummary includes latency percentiles (OBS-01) via _latency_percentiles.
- qval/engine/pricing.py: load_pricing(config), compute_cost(model, prompt_tokens, completion_tokens, pricing). Prices live in config/model_config.json "pricing" map (USD per 1k tokens); unknown model -> warning path (reset_warnings exists for tests).
- qval/scorers/: base_scorer.py holds detector FUNCTIONS + phrase libraries (REFUSAL_PHRASES, SAFE_ALTERNATIVE_PHRASES, ...) + get_scorer(category) dispatcher. EXACTLY 7 categories, one scorer module each: safety, instruction_following, bias, toxicity, hallucination, robustness, privacy. There is NO class-based detector registry: ScorerFn = Callable[[TestCase, str, str|None], dict] returning {"status", "score", "scoring_reason", "detector_results"}.
- qval/gate/: diff.py diff_runs(baseline: CanonicalRun|None, current: CanonicalRun) -> RunDiff (with Regression, CategoryDelta); decision.py evaluate(diff, thresholds: GateThresholds|None) -> Decision. GateThresholds is the ONLY policy mechanism today; qval/templates/policy.yaml (release_policy: block_on / pass_rate_floor / require_review) is scaffolded by init but NOT parsed by any code yet.
- qval/importers/: base.py BaseImporter, registry.py (register / get_importer / available_tools — CLI choices derive from it). Only PromptfooImporter is registered. NO DeepEval importer.
- qval/reports/: canonical_report.py is the F-05 report (render_markdown(run, diff, ...), render_html(run, diff, ...)); report_generator.py + html_template.py are the OLDER native-report path still used by qval run.
