🕒 The web app (hosted Evidence Room): build it early? — No. Post-PMF.
The wedge is the OSS artifact + verify (CLI + static HTML passport) — that drives adoption and demos in 10 seconds with no server. The hosted web app is the monetization layer, not the wedge.
  • Why wait: multi-tenant hosting + RBAC/SSO/retention before you know what reviewers need = burning runway pre-PMF. Giskard Hub and Promptfoo enterprise already ship dashboards — competing on "dashboard" early = a weaker clone.
  • What covers MVP instead: the static HTML passport opens in any browser — reviewers read findings + decision with zero infra.
  • Earliest defensible web step (optional, OSS): a thin local review viewer (qval serve) reading local files — nicer than static HTML, still no hosting. Must not jump ahead of the flagship.
  • Trigger to start the hosted app: real OSS pull — stars, repeat passport creation, teams asking for shared history & sign-off.
Do not build before OSS PMF. No hosted product until the OSS core shows traction.
